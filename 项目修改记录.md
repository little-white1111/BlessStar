## 契约绑定字段说明（第17天 D 方案起）

当日变更涉及 **可验收边界**、**Gate 脚本**、**`docs/contracts/**`** 或 **`docs/gates/gate_registry.json`** 时，在每条变更记录下增加 **契约绑定**（无则省略）。机器布局与步骤以 `docs/contracts/index.json` 为准。

**推荐写法（一行或分号分隔）：**

```text
- **契约绑定**: <契约ID> / <GATE-ID> / <脚本路径或 ctest 命令摘要>；✅ validate + compile [+ gate_runner | day17 | integration | regression]
```

| 片段 | 含义 | 示例 |
|------|------|------|
| 契约 ID | 实例条文 | `C-ST-1`、`C-IX-4` |
| GATE-ID | `gate_registry` 中的门禁 | `GATE-STYLE-PREFIX` |
| 脚本/命令 | `implementations[].entry` 或 gate `command` | `tools/scripts/gates/check_public_api_prefix.py` |
| 校验结果 | 子任务B 已执行的契约门禁 | `✅ validate + compile`；改 gate 行为时加 `gate_runner` 或 `ctest -L day17` |

**注意：** `rule` 字段仅作文条语义；**可执行依据**为契约实例中的 `implementations[]` 与 `gate_registry` 中的 `command`。改 gate 定义后可运行 `python tools/scripts/maintenance/sync_contract_implementations.py` 同步 `implementations[]`。
