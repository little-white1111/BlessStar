3. **输出规范**：完成当日所有既定任务后，提交一份简洁的完工报告：

4. **历史隔离**：开展下一日工作时绝不读取任何历史操作记录或过往工程成果，只基于最新内核源码和主任务最新决策。
5. **回滚处理**：仅当收到"⏪ 回滚指令"时，才读取对应日期的备份，完成源码复原后立刻关闭历史读取。

## 测试与性能评估规则

### 测试执行流程
1. **测试设计**：以当日子任务A的功能设计文档为依据，设计覆盖所有功能点的测试方案
2. **测试代码编写**：编写临时测试代码，覆盖边界条件、正常流程、异常情况
3. **测试执行**：在后台执行测试回归，验证代码实现正确性
4. **问题修复**：测试失败时，立即定位并修改源代码，直到所有测试通过
5. **测试代码清理**：用户确认当日代码达标后，删除临时测试代码

### 性能评估流程
1. **二次评估**：测试通过后，从当日子任务A的总体功能链路角度进行性能评估
2. **多角度评估**：包括响应时间、内存占用、CPU使用率、并发处理能力
3. **评分展示**：向用户展示性能评分表
4. **优化循环**：用户否决时，设计性能优化方案，遵从用户建议进行定向优化
5. **确认完成**：用户确认当日代码完成后，才能结束当天任务
